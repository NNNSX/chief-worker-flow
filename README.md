# Chief Worker Flow Skill

这是一个用于 Codex 的通用任务流管理 skill。它把主会话视为 `chief`，把一次性子会话视为 `worker`，用于处理复杂、多步骤、多模态、大文件、资料检索、批处理和并行审查任务。

## 设计目标

- 主会话保持轻量，只保留用户意图、决策、路径、链接、简短观察和最终汇报。
- 图片、视频、音频、provider 调用、大文件处理、资料检索等重任务交给有边界的 worker。
- worker 完成后返回证据和结论，然后停止，不长期占用上下文或进程。
- 对可并行的独立分支，先并行派发，再统一等待和汇总。
- 对高风险任务，分派时写清验收标准、失败条件和证据要求，避免只凭 worker 概括性判断。

## 适用场景

- 多模态任务：图片审查、视频抽帧、音频转写、OCR、mask、overlay、视觉对比。
- provider 调用：AI 生图、修图、音视频生成、连接性测试、批量生成测试。
- 文件处理：大目录梳理、批量转换、批量重命名、日志分析。
- 资料检索：多个来源并行查证、官方文档对比、当前事实验证。
- 复杂审查：多个候选、多处代码、多份文档、多个风险点的独立检查。

## 使用方式

在 Codex 中触发：

```text
使用 $chief-worker-flow 协调这个任务……
```

典型用法：

```text
使用 $chief-worker-flow 检查这几张候选图，但不要让主会话加载图片。
```

```text
使用 $chief-worker-flow 并行查三份官方资料，最后只汇总关键结论和链接。
```

## 核心规则

- 主会话负责理解、分派、收集、验证和汇报。
- worker 负责有边界执行，并返回简短报告。
- 媒体内容默认不进入主会话。
- worker 报告是证据，不是自动真理；关键约束必须有可验证证据。
- 不越过用户当前授权阶段，例如草图不自动进入最终生成，候选不自动收纳。
- worker 完成、阻塞或不再需要后必须关闭。
